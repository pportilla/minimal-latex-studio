# Arch Package Maintainer Review Request

Package: `minimal-latex-studio`

Upstream: `https://github.com/pportilla/minimal-latex-studio`

Summary: Native Qt 6 LaTeX editor with local compiler integration, Poppler PDF
preview, SyncTeX source/PDF navigation, and embedded QTermWidget terminal.

## Why It Fits Arch

- Native C++/Qt application with no bundled binary dependencies.
- Uses existing Arch packages: `qt6-base`, `poppler-qt6`, `qtermwidget`,
  `hicolor-icon-theme`, `gcc-libs`, and `glibc`.
- TeX tools are detected at runtime and kept as optional dependencies.
- CMake build has no network access and installs only expected files under
  `/usr/bin` and `/usr/share`.
- Package includes `check()` validation for desktop and AppStream metadata.
- Arch package sources are prepared with explicit MIT/REUSE metadata.

## Validation Performed

```bash
cmake --build build-native -j2
ctest --test-dir build-native --output-on-failure
clang-tidy src/main.cpp -p build-native --removed-arg=-mno-direct-extern-access --quiet
desktop-file-validate resources/applications/io.github.pportilla.minimal-latex-studio.desktop
appstreamcli validate --no-net resources/metainfo/io.github.pportilla.minimal-latex-studio.metainfo.xml
namcap packaging/arch/PKGBUILD
makepkg -C --noconfirm
namcap minimal-latex-studio-*.pkg.tar.*
```

Expected current `namcap` package output:

```text
Dependency libstdc++ detected and implicitly satisfied
Dependency libgcc detected and implicitly satisfied
Dependency included, but may not be needed ('gcc-libs')
```

`gcc-libs` is kept intentionally for a C++ Qt binary.
